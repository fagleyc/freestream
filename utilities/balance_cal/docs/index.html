Overview

Balance Cal (balcal_gui) is the Python port of the MATLAB FB_Cal_GUI/ForceCal.mlapp calibration app. It acquires bridge voltages from an analog-input DAQ while dead-weight loads are applied to a wind-tunnel balance, walks the operator through all twelve loading orientations with guide images, fits and diagnoses the calibration matrix, and writes a "Voltage Calibration File 3.1" (.vol) — the same format consumed by the device drivers' balcal.read_vol_file, the freestream Forces monitor, and Streamlined.

# standalone (NI USB-6351 default)
python run_balcal_gui.py
python run_balcal_gui.py --sim                  # no hardware
python run_balcal_gui.py --backend strainbook   # StrainBook/616

It can also be launched from freestream via Advanced → Balance Calibration…, in which case it shares freestream's live DAQ stream when connected (backend/sim/device/balance-type controls lock to the shared device).

  • Simulation: check "Simulate (no hardware)" (or --sim) to run against the driver simulator — the status lamp shows SIMULATION instead of ACQUIRING. All tests run this way.
  • Shared device: when embedded in freestream the already-connected driver is reused; acquisition adds any software tare back so freestream's tare never leaks into calibration data.
  • Excitation: measured from the Excitation channel by default, or set a constant supply voltage ("Constant" mode, or File → Set constant excitation for session… to overwrite every stored point).

Calibration Procedure

1 · Measurement Setup tab

  1. Pick backend, device, scan rate (default 1000 Hz), and balance type.
  2. Edit physical channels / ranges in the channel table before connecting.
  3. Fill in Balance Info: max load and distance to balance center [in] per element, operator ("Cal'd by"), serial number, outer diameter, date.
  4. Press Connect — the lamp turns ACQUIRING (or SIMULATION).

2 · Orientations and moment arms

The Calibration Procedure tab steps through 12 orientations (positive and negative per element, e.g. N1_pos … Mx_neg). Selecting one shows the matching rig photo (FB_* / MB_* guide images) and sets the moment arm:

  • Direct forces: arm = 1.0 (load = dead weight).
  • Moment-balance pitch/yaw: arm = d1 + d2 (station-to-station distance; weight hangs at the opposite station). A warning appears if the paired distances are not entered — the arm silently falls back to 1.0 otherwise.
  • Roll: uses the roll arm distance (default 2.0 in).

Applied load = dead weight × moment arm. Loads exceeding the entered max load prompt for confirmation.

3 · Acquiring test points

  1. Hang the weight, let the balance settle, press Acquire Test Point — [orientation] and enter the dead weight.
  2. The app averages the six bridges + excitation over the chosen window (off the GUI thread), stores per-channel standard deviations, and warns if excitation is below 0.5 V.
  3. Always include a zero-load point per orientation — the .vol consumers use load = 0 rows for zero-offset correction.
  4. The live cal plot shows load vs primary bridge volts with ±1σ error bars; noisy points are highlighted and a dashed linear trend is drawn. The Time History tab shows the last acquisition per channel.

4 · Fit and diagnostics (Cal Summary tab)

  • Choose the fit type (Linear / Quadratic / Cubic) and press Compute Calibration. The fit runs through the exact consumer code path (balcal.read_vol_file + calc_coeffs on a temp .vol), reporting per-element R², RMS bias, and the calibration matrix.
  • The diagnostics engine adds iterative MAD-based outlier rejection (masking-proof), per-section health (missing zero-load rows, offset-suspect sections), excitation anomalies, off-diagonal channel scans (Theil-Sen trend + robust z), cross-talk burden notes, and a leave-section-out influence scan — rendered first in plain language (WHAT / WHY / FIX), then in full detail.
  • Interactive views: Applied vs Predicted, Residuals vs Applied, Channel volts (off-diagonal). Suspect points can be Repair[ed] from Trend, Edit[ed], Exclude[d] & Refit, Delete[d] & Refit, or jumped to in the table; excluded points are left out of both the fit and the written .vol.
  • Save Report… writes the text summary.

5 · Writing (and reloading) the .vol

  • Write to File… / File → Write .vol… validates the session first (missing max loads, distances, operator, serial are listed) and writes ASCII — encoding happens before the file is opened, so a bad character can never truncate an existing calibration.
  • File → Load .vol for editing… round-trips an existing file back into the session (append more points, re-fit, re-write byte-identically).
  • File → Summarize existing .vol… reports any .vol without loading it for editing.

.vol File Format ("Voltage Calibration File 3.1")

Voltage Calibration File 3.1
Date--> 6/6/2025
Calibration performed by--> ...
[Balance Description]
Balance Type--> 5 Force/1 Moment
Balance Serial Number--> ...
Balance outer Diameter--> 0.75 in.
[Maximal Balance Loads]
N1--> 50 lbs
...
[Distances]
Distance from N1 to center of Balance x1 in inches--> ...
...
[N1 pos]
Number of Loads--> 5
Load[lbs], N1[Volt], N2[Volt], Y1[Volt], Y2[Volt], Ax[Volt], Mx[Volt], Exct.[Volt]
0, 1.0000000000E-03, ...
  • One [Element pos|neg] section per orientation; volts in %.10E scientific notation, loads plain.
  • Excluded points are skipped on write; per-point standard deviations are not part of the format (reloaded points carry none).
  • Consumers fold zero offsets from load = 0 rows and normalize by the excitation column, then least-squares fit Linear (6×6), Quadratic (12×6), or Cubic (18×6) matrices.

Troubleshooting

SymptomCause / fix
No NI devices foundCheck NI MAX alias and cabling; press "Search" again, or run with --sim to work without hardware.
Excitation warning (< 0.5 V)Excitation channel not wired or supply off. Wire it, switch excitation mode to "Constant", or use File → Set constant excitation for the whole session.
Moment arm warningThe paired station distances (x1/x2 or y1/y2) are missing from Balance Info — enter them, otherwise the arm silently falls back to 1.0.
Poor R² on one elementOpen Cal Summary diagnostics: look for flagged outliers, a missing zero-load row, or an offset-suspect section; repair or exclude-and-refit.
Off-diagonal channel anomalyA bridge shows unexpected response to another element's loading — usually a swapped channel or mechanical interference; the "Channel volts (off-diagonal)" view pinpoints the row.
.vol write fails with encoding errorA non-ASCII character in a metadata field; the file is never truncated — fix the field and re-write.
Old tab-delimited V1 file won't loadOnly "Voltage Calibration File 3.1" comma-separated files round-trip; re-create the calibration or convert the file.
Controls greyed out when opened from freestreamExpected: the shared live stream owns backend/device/balance-type; disconnect in freestream to change them.
