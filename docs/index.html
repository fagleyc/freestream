Overview

Freestream is the tunnel-side control and acquisition suite — the counterpart to Streamlined (the reduction app). It commands attitude and tunnel-speed sweeps over the facility's device drivers, streams live engineering-unit monitors, and records raw, self-describing per-point run files that Streamlined reduces.

Core principle: calibration is never applied at capture. Balance channels are recorded verbatim (volts for internal balances, resolved loads for the external balance); .vol calibration files are recorded as pointers in metadata only. Anything derived on-screen (Mach, q, wind-axis forces) is display-only and never persisted as reduced data.

Layers: hal (capability contracts) → adapters (thin wrappers over the device drivers) → manager (registry/modes) → sweep (state-machine engine) → recorder (HDF5/.mat/.xlsx) → app (dock GUI). Launch with python run_freestream.py.

Architecture

HAL — capability roles (freestream/hal.py)

Devices are described by runtime-checkable capability protocols, not by type:

ProtocolContract
DeviceBaseid, label, connect()/disconnect(), connected, sim, status() (OK / OFFLINE / FAULT).
Streamingstart()/stop(), channels() (each a ChannelSpec: name, unit, HDF5 group, kind), latest(), drain_block(), sample_rate().
Positionermove_to(**axes), axes() (AxisSpec: name, unit, limits, tolerance), positions(), settled(), stop_all() (E-STOP path).
Zeroablezero(seconds) — balance tare.
SetpointDeviceset_target(**kw), at_target(), readback(). The tunnel PLC exposes fan RPM, not Mach — Mach/q are computed in derived.py.

Adapters (freestream/adapters/)

One thin wrapper per driver; drivers themselves are never modified. Each adapter exposes its driver config (config_dict()/apply_config_dict() — excluding force_sim, which the manager owns) and its native settings dialog.

Device manager (freestream/manager.py)

  • DeviceManager(mode, sim, ...) builds the registry from devices_manifest.json, wiring role → adapter per mode (roles: positioner, balance, tunnel_conditions, tunnel).
  • connect_all() / disconnect_all(); stop_all_motion() is the E-STOP path (stops every Positioner).
  • record_blockers() implements refuse-to-record: any device not OK, plus registered extra blockers (e.g. the Forces page's balance-overstress interlock), blocks sweep start/recording.
  • DeviceManager.custom(device_ids) builds an explicit device subset ("custom" mode); roles are inferred from capabilities.

Sweep engine (freestream/sweep.py)

SweepEngine is a per-point state machine (no Qt) with states queued → moving → acquiring → done / failed / skipped. Per-point cycle: refuse-to-record check → set tunnel → move positioner → wait settled → optional zero → drain stale samples → dwell → acquire → write → advance.

  • Pause/Resume take effect at the next point boundary — the current point finishes acquiring and writing first. A failed point auto-pauses the sweep with the remaining points re-queued.
  • Abort is graceful; E-STOP calls stop_all_motion() immediately, then aborts.
  • Tunnel gate: with tunnel control disabled (default) and Mach-verify enabled, a monitor-only operator wait dialog opens at each Mach change; with control enabled, the Mach loop commands fan RPM (a run-sheet rpm column bypasses the Mach loop verbatim). With Mach-verify off the gate is skipped entirely.
  • Air state is derived from Mach: mach 0 → AirOff, else AirOn.

machloop.py maps target Mach to clamped fan RPM (rpm_per_mach), verifies against measured isentropic Mach within mach_tolerance, and iterates proportionally up to mach_max_iterations before faulting (RPM-proxy fallback in SIM).

Recorder (freestream/recorder.py)

Hdf5Recorder writes one file per test point under <data_root>/<config_name>/ in the selected output_format (h5 / mat / xlsx; dependency checked at construction). Run numbers are generic (run_NNNN), continuing across all formats. See Data Files.

Derived quantities (derived.py, aero.py)

  • derived.py is the one isentropic Mach/q module (identical constants to Streamlined's SSWT chain: γ=1.4, R=287.058, psi→Pa 6894.75729). Tunnel condition channels Pdiff / Ptot / Temp are found by name across streaming devices (SWT: one DaqBook; LSWT: split across NI + Heise). tunnel_state() never raises; it flags validity.
  • aero.py does display-only balance reduction for the Forces page: .vol calibration → wind-axis Lift/Drag/Side/Roll/Pitch/Yaw → coefficients over q·S with utilization vs rated loads (overstress at ≥100%). Never persisted.

GUI Walkthrough

Dock layout: top command bar; left Devices rail (one status card per device: status pill, SIM badge, live readout, right-click Configure/Connect/Disconnect); center monitor tabs (Tunnel, Balance, Position, Forces, Results); right Sweep Planner; bottom Run Log. Slide-out pane handles: Ctrl+1 device rail, Ctrl+2 planner. Esc is E-STOP.

Tunnel dashboard (Tunnel tab)

  • Top band: fan rotor + RPM gauge (reused from the tunnel_plc app), α/β attitude crosshair pad with ghost target marker, and stat tiles — Mach, q, Fan RPM (accented), P total, T total, Velocity.
  • Status lamps: fan running, inverter fault, oil level low, bearing temp low, bearing heater on, console control.
  • Three linked strip charts (Mach / q / Fan RPM, ~120 s of history at 5 Hz).
  • Responsive: lamp column, fan graphic, and attitude pad drop out as the window narrows.

Forces page

  • Six wind-axis tiles — Lift, Drag, Side [lb]; Roll, Pitch, Yaw [in·lb] — each with a peak-hold load bar: filled bar = live utilization of rated load, bright marker = rolling 30 s peak (reset on tare), tick at 100%.
  • Device-owned .vol: the calibration file, fit type, and Force/Moment layout are owned by the balance device's configuration and re-read each tick — the "Balance device…" button jumps to that device dialog. There is no separate Forces-page calibration state.
  • Internal balances: live volts → .vol calibration → wind-axis loads and coefficients (display only, using a non-consuming tail of the stream so the recorder never loses samples). External (ATE) balance: resolved loads compared directly to rated limits.
  • Overstress interlock: any channel at ≥100% utilization raises an alarm and registers a record blocker — sweeps refuse to record until it clears.

Sweep planner

  • Axis fields adapt to the mode: α/β/M (aero modes) or X/Y/Z (traverse). Each cell takes the sweep grammar.
  • Build Grid expands the cells into the point table (Mach outermost with the air-off zero point first, then beta, alpha innermost); statuses colorize live during the run; double-click a FAILED row to re-run just that point.
  • Import Run Sheet loads a run-book workbook (or flat CSV/XLSX fallback with columns alpha, beta, mach, x, y, z, dwell_s, samples, air_state, rpm — unknown columns ride into file metadata verbatim).
  • Dwell and samples-per-point come from Measurement Setup (run sheets can override).

Sweep grammar (sweepgrammar.py)

CellMeaning
(blank)Axis not swept.
2Single value.
0,2,4Comma list.
-4:2:8Range start:delta:end — the middle value is the step, end inclusive → -4,-2,0,2,4,6,8. Partial tail is dropped; delta sign follows the start→end direction.
0:2:10RReturn sweep: up 0…10 then back down (apex not repeated); points are tagged up/dn and filenames get _up/_dn when hysteresis tagging is on.
@nameNamed array from the run-book's Named Arrays sheet.
csv:aoa.csvOne column from a CSV file.
-4:2:8, 10, 12Any of the above, comma-joined.
The middle number is the delta, not the end — this replaced the old start:stop:step reading. A Mach sweep automatically prepends the air-off Mach 0 point (planner path).

Run-book import

The primary import path is the 5-sheet workbook (Freestream_RunSheet_Template.xlsx): Guide, Test Info (test/model/engineer/operator, reference dims Sref, cref, bref, MRC_x/y/z, objectives…), Run Matrix (one row per run: enable flag, alpha/beta/mach grammar cells, samples, rate, model config, notes), Model Configs (named configuration rows — every column rides verbatim into point metadata), and Named Arrays (definitions for @name). The run-sheet dialog previews everything, shows live expanded point counts, and loads one run or all enabled runs. Loading a run merges Test Info + reference dims into the configuration and sets the output folder from the run's Model Config.

Device dialogs (embedded apps)

Configure… on any device opens DeviceConfigDialog: the device's own standalone app panel (CrescentPanel, StrainbookPanel, DaqbookPanel, AteBalancePanel, TunnelPanel, TraversePanel, LswtPanel, StingPanel, NiDaqPanel, HeisePanel) is embedded as the primary tab and shares the live driver — one device, one connection. Settings/axis tabs edit the driver config (suite-owned fields like the streaming rate are hidden); Apply rebinds a running driver; Save/Load to file supported. Tunnel-control panels are ARM-gated.

Advanced ▸ Balance Calibration

Opens the balance_cal (.vol acquisition) window — one window per session. When freestream is connected with the StrainBook as the balance, the window shares the live driver; otherwise it runs standalone with its own backend selection. See the balance_cal documentation (utilities/balance_cal/docs/index.html).

Data Files

One file per test point, under <data_root>/<config_name>/:

run_{NNNN:04d}[_alpha_{a}][_beta_{b}][_mach_{m:.2f}][_x_..][_y_..][_z_..][_up|_dn].h5
e.g.  run_0012_alpha_2.0_beta_0.0_mach_0.30.h5
  • Run numbers are generic and monotonically increasing across all formats (never model-specific). Air state is a root attribute, not a filename token — a mach_0.00 point already marks air-off.
  • A custom filename_template may reference {run}, {air_state}, {dir}, {config_name} and any point-meta key.

HDF5 layout

/ (root attrs: run_number, timestamp, mode, config_name, operator, air_state,
   positions_source, balance_group, balance_type, span_config, + run-sheet meta)
/StrainBook_0 | /ATE_Balance | /NI_USB_6351   raw balance channels (verbatim)
/DaqBook2005 | /Heise                          tunnel-condition channels
/Positioner                                    Alpha/Beta (or X/Y/Z)
/Tunnel                                        RPM_meas, RPM_cmd, Mach_cmd,
                                               Mach_meas*, q_meas*   (*derived)
/Time/Time
/meta/devices/<id>                             device metadata (cal_file = POINTER)
/meta/config                                   full config JSON

Each dataset carries wf_increment, wf_samples, wf_start_time, and unit attributes (mirroring TDMS waveform conventions).

Output formats and Streamlined interop

FormatNotes
HDF5 (.h5, default)Read directly by Streamlined (read_hdf5_file); layout mirrors the TDMS group structure it reduces.
MATLAB (.mat, v5)Struct per group + meta (run attrs, channels, devices, config JSON, sanitized-name map); read by Streamlined's read_mat_file.
Excel (.xlsx)Review only (one Data sheet per group + Meta/Channels/Devices/Config sheets) — not readable by Streamlined.

Derived Mach/q use the identical isentropic constants as Streamlined's SWT chain, and reference dims (Sref/cref/bref/MRC) from the run book are recorded so Streamlined can reduce coefficients without re-entry.

Troubleshooting

SymptomCause / fix
Start Sweep refused, red banner Refuse-to-record: a device is OFFLINE/FAULT or the balance is overstressed. The banner lists every blocker; clear them (connect/reset/tare) and retry.
"No data devices" info banner The active mode has no Streaming device (e.g. SWT-Traverse) — automated sweeps are disabled by design; manual positioning still works.
Tunnel never reaches Mach / dialog every point Tunnel RPM control is off by default (Red Lion rejects Block2 writes) — freestream is monitor-only and asks the operator to set fan speed. Enable "Tunnel RPM control" in Measurement Setup only if the PLC accepts writes; disable "Verify Mach at each point" to skip the gate entirely.
Mach loop FAULTs after N iterations rpm_per_mach is mis-tuned for the rig, or tunnel conditions channels (Pdiff/Ptot/Temp) are missing/invalid. Check the DaqBook/Heise cards and the dashboard tiles.
StrainBook streams wrong values / excitation stuck at 10 V in LIVE A defaults bundle saved during a SIM session used to re-arm force_sim on the live driver (rig-found 2026-07-22). Fixed — SIM/LIVE is owned by the manager and excluded from saved device configs. Re-save defaults if they predate the fix.
Forces page shows "no cal" The .vol path/fit/layout are owned by the balance device config — set them in the balance device panel (button on the Forces page), not in Measurement Setup.
Old external-balance files show N1/N2/… channels Files written before ATE truth-naming; readers (incl. the Results panel) fall back to the legacy alias map. New files record true Lift/Drag/… channels.
Output format change seems ignored The recorder is rebuilt at sweep start — a format change applies to the next sweep. Missing scipy (mat) or openpyxl (xlsx) is reported at that point and falls back to HDF5.
Sweep paused unexpectedly A point failed (move timeout, device fault, empty acquisition) — the sweep auto-pauses with remaining points re-queued. Fix the cause, then Resume; or double-click the FAILED row to re-run it.
Legacy config won't load / mode name unknown mode1/2/3 are auto-migrated; unknown keys are dropped on load. Re-save the config to modernize it.
