StrainBook/616 — Balance Bridge DAQ

User documentation for the strainbook_616 package — Python driver + dark-mode PyQt6 GUI for the IOtech StrainBook/616.
Author: C. Fagley — casey.fagley@afacademy.af.edu

1  Overview & purpose

The IOtech StrainBook/616 at the USAFA subsonic tunnel is the internal-balance bridge readout: six strain-gage bridges — N1, N2, Y1, Y2, Axial, Roll (the Streamlined channel names) — plus an Excitation readback channel. This package is a fully standalone driver + GUI and part of the AeroVIS suite; the same panel embeds unchanged in the Freestream host with a single shared driver.

Bridge channels display input-referred volts; recorded data is always raw volts. Conversion to forces happens either live in the app's Forces tab (via a Streamlined .vol balance calibration) or downstream in Streamlined/AeroVIS.

Architecture facts (verified live 2026-07-06 / 2026-07-16)

These are hard-won rig facts recorded in the package README — do not re-learn them:

  1. Channel map: the built-in strain conditioning is an internal WBK16 bank on DaqX channels 9–16; front-panel "CH n" = DaqX channel n + 8 (STRAIN_CHANNEL_OFFSET). Channels 0–8 are the raw WaveBook core (module type Wbk516A) and reject all Wbk16 options with error 67. LabVIEW's channel table hides this offset.
  2. Data is SIGNED int16 regardless of the DafUnsigned scan flag — 0 counts = 0 V, 0x7FFF = +FS, 0x8000 = −FS. An offset-binary decode maps real ~100 µV bridges onto the ±range rail (the 2026-07-16 "all channels railed" bug).
  3. Scan polarity is per channel and it matters: bridge channels need DafBipolar — scanned unipolar they peg at 0x8000. The 0–10 V excitation channel scans unipolar, which shifts the data one half-span down (0 V input = −FS): true volts = decoded + 5.
  4. Options download at arm time, only for channels in the scan list — configure options on 9–16 and scan 9–16.
  5. OutSource=ReadExcVolts monitors only the INTERNAL excitation banks. With the rig's EXTERNAL supply it reads 0 V. The external excitation is instead wired into CH8's differential input and measured as a plain voltage (OutSource=ReadSignal, full bridge 4-wire, ×1, unipolar) → reads the true supply (9.86 V measured). The internal excitation banks are NEVER commanded — two sources on the excitation bus would fight.
  6. Open channels rail at ±range (e.g. ±11.19 mV at ×447) — the expected wind-off-unplugged signature, matching LabVIEW.
  7. SSH defaults OFF pending verification with a connected balance (DafSSHHold = 0x10 exists as a per-channel scan flag — the likely missing piece if SSH is wanted later).
  8. Single-session device: a dead client wedges the session. The status page http://192.168.1.123/ shows Opened; its Reset Device link clears it.

Live-verified end to end 2026-07-16 with the balance cabled and a 10 V external supply: 200 Hz × 7 channels, bridges at −250/−200/−695/+764/+250 µV (Axial +16.5 mV genuine offset), excitation 9.864 V — matching LabVIEW.

3  Getting started

# from the devices/ directory
python run_strainbook_616_app.py --sim   # no hardware
python run_strainbook_616_app.py         # real StrainBook_0 (192.168.1.123)
python probe_strainbook.py               # 3-step live probe

First-connect checklist

  1. The wavebk kernel service must be RUNNING (re-run check_daqx_driver.ps1 after major Windows updates — same Windows 11 driver-policy caveat as the DaqBook, see daqbook_2000/README.md).
  2. The DaqX device alias StrainBook_0 must exist (Daq applet).
  3. Confirm nobody else holds the session — check http://192.168.1.123/ (should not say Opened; use Reset Device if it does).
  4. Launch the app, check the alias and scan rate in the Connection bar, press Connect. The lamp shows ACQUIRING @ 200 Hz (or SIMULATION).
  5. With the tunnel off, expect small bridge signals (hundreds of µV) and excitation ≈ the external supply voltage. Open/unplugged channels rail at ±range — that is normal.

Live tab

  • Stat tiles — one per enabled channel, smoothed over the tile-averaging window.
  • Tare (zero bridges) / Clear tare — software tare of the bridge channels.
  • Window combo (10 s / 30 s / 2 min / 5 min), Pause/Resume, and Follow live (re-pins the time axis after zoom/pan).
  • Show row — one colored checkbox per channel toggling its curve on the history plot (the excitation strip collapses when its channel is hidden).
  • History plot — all bridges overlaid on one mV plot with a slim excitation strip below; redraws are envelope-decimated so long windows and high rates stay smooth.

Forces tab

Live balance forces/moments from a Streamlined .vol calibration: load the file, choose Linear / Quadratic / Cubic, and every UI tick the ring buffer's bridge voltages run through the Streamlined pipeline (volts/excitation → elements → body-frame Fx…Mz). Per-element loads are compared against the balance's rated maxima with amber/red overstress warnings and rolling peak-hold utilization bars. The Force ↔ Moment balance-layout selector renames the four bridge channels (N1/N2/Y1/Y2 ↔ AftPitch/AftYaw/FwdPitch/FwdYaw) on the live device.

Channels tab

Mirrors the rig's LabVIEW "Additional StrainBook Parameters" table. Edits apply to the config immediately; a (re)connect pushes them to the hardware. Columns: On, CH, Name, Bridge, Range, Filter, AC, Inv, SSH, Offset, Unit. Setting a channel's Range to "0 to 10 V" turns it into the external-excitation readback (CH8). There are no internal excitation-bank controls — the external supply is the sole authority.

6  Safety & limits

  • This app commands no motion — it is measurement-only. The balance protection features are software limits.
  • Overstress warnings (Forces tab): per-element loads from the .vol calibration are compared against the balance's rated maxima — amber above warn_utilization (default 80 %), red at/over 100 %, with a status-bar "⚠ BALANCE OVERSTRESS — reduce load!" message while active.
  • Excitation policy: the internal excitation banks are never commanded; the external supply is the sole authority. CH8 only reads it back.
  • Session hygiene: always disconnect via the GUI (or Ctrl-C) — killing a client mid-acquisition wedges this single-session device.

7  Troubleshooting

SymptomCause / fix
All channels railed at ±full-scale Offset-binary (unsigned) decode of the signed int16 data, or bridge channels scanned unipolar. The driver decodes signed and scans bridges bipolar — check custom scripts against counts_to_volts.
One open channel rails at ±range (e.g. ±11.19 mV) Normal wind-off-unplugged signature, matching LabVIEW.
Excitation reads 0 V though the supply is on OutSource=ReadExcVolts monitors only the internal banks. The rig's external supply is read via CH8 as a plain voltage ("0 to 10 V" range) — verify CH8 is enabled and configured for excitation readback.
Excitation reads ~5 V low The unipolar scan shifts data one half-span down; true volts = decoded + 5. The driver's read_excitation branch applies exactly this.
Wbk16 options rejected with error 67 Options sent to DaqX channels 0–8 (raw WaveBook core). Strain options belong on channels 9–16 (front-panel CH n = DaqX n + 8), and only channels in the scan list accept them.
Connect fails / device busy Single-session device: another client (or a stale session) holds it. Status page http://192.168.1.123/ shows Opened — use its Reset Device link.
Driver service not running after a Windows update Windows 11 driver policy blocks the 2006 IOtech kernel drivers. Re-run check_daqx_driver.ps1; the wavebk service must be RUNNING (see daqbook_2000/README.md for the full procedure).
Dead (never-sampled) channels with SSH on SSH gave dead channels in live bring-up — leave it off (default) until the SSH scan-slot behaviour is verified. The 1 kHz filter + 200 Hz scan keeps inter-channel skew negligible meanwhile.
Reproduce the verified raw-count table python diagnose_raw_counts.py prints raw hex + decoded volts per channel for rig debugging.
