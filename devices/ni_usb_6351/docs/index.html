NI USB-6351 — Analog I/O DAQ

User documentation for the ni_usb_6351 package — driver + PyQt6 dark-mode GUI for the NI USB-6351 (X series) multifunction DAQ wired to the rig's force balance.

1 · Overview & purpose

One NI-DAQmx analog-input task performs hardware-timed continuous sampling into the DAQmx input buffer, drained by a poll thread (AnalogMultiChannelReader). The task's start trigger is configurable — immediate, digital edge on a PFI line, or analog edge on APFI0 / a scanned AI channel; with a hardware trigger the task arms at connect and the first samples arrive only after the edge. A second on-demand task drives the two analog outputs with static DC levels, or a hardware-timed regenerated waveform.

Bridge channels record raw volts (scale forced to 1.0); converting to forces is done via the balance .vol calibration (a self-contained copy of Streamlined's calibration math — same file format, same results): read_vol_file → least-squares cal matrix → bridge volts / excitation → element loads → body-frame Fx, Fy, Fz, Mx, My, Mz using the balance distances. Units follow the .vol file (lb / in·lb at the subsonic tunnel).

2 · Hardware & connection

Six balance bridge outputs (via external signal conditioning) on AI0–AI5 with the Streamlined names, excitation readback on AI6, and a spare tunnel channel on AI7 (disabled until wired):

Native AI ranges are ±0.1/0.2/0.5/1/2/5/10 V — the smallest native range covering the requested span is used. Terminals: DIFF, RSE, NRSE.

Balance layouts (Force ↔ Moment)

The four bridge channels are physically identical for both layouts — only their NAMES differ (Streamlined's reducer keys on the exact names). Switching the layout is a pure rename: Force → N1/N2/Y1/Y2, Moment → AftPitch/AftYaw/FwdPitch/FwdYaw. A live device keeps its ring-buffer and tare keys in sync through the rename.

Outputs and triggers

  • AO0/AO1 (±10 V): static DC level, or a regenerated sine/square/triangle waveform (amplitude, frequency, offset; ao_update_hz sample clock, default 10 kHz).
  • Start trigger: immediate; digital_edge on PFI0–PFI15 (rising/falling); analog_edge on APFI0 or a scanned AI channel crossing level_v. Continuous acquisition supports a start trigger only — no reference/pre-trigger.

Connection

The device is addressed by its NI-MAX alias (default Dev2). Sim mode (force_sim) swaps in a synthetic signal core — no NI-DAQmx install, no hardware.

3 · Getting started

python run_ni_usb_6351_app.py --sim   # synthetic signals, no hardware
python run_ni_usb_6351_app.py         # real USB-6351 (NI-MAX alias Dev2)
python -m pytest tests\test_ni6351.py

First-connect checklist

  1. Confirm the device alias in NI-MAX matches the Device field (default Dev2).
  2. Set the per-channel sample Rate (default 1000 Hz), then Connect — acquisition starts immediately (or arms, if a hardware trigger is configured; the lamp shows ARMED @ … Hz until the edge arrives).
  3. Load the balance .vol calibration on the Forces tab (auto-reloaded at startup once configured).
  4. Tare (zero bridges) with the model unloaded before taking force data.

4 · GUI guide

Connection bar

Device (NI-MAX alias) · Rate (Hz) · Simulate · trigger-state readout (trig: PFI0 / ARMED — PFI0 / Triggered) · status lamp (DISCONNECTED / SIMULATION / ARMED / ACQUIRING @ actual Hz) · Connect / Disconnect.

Live tab

Channel tiles (averaged over tile_avg_ms), Tare (zero bridges) / Clear tare, plot window selector (10 s – 5 min), Pause/Resume, Follow live (re-pins the time axis after zoom/pan), and the multi-channel history plot fed from the device-owned ring buffer.

Forces tab

Load/inspect the balance .vol calibration (balance type/serial, cal type Linear/Quadratic/Cubic, R²), pick the balance layout (Force ↔ Moment — renames the bridge channels everywhere), and watch live body-frame forces/moments with per-component utilization bars against the balance max loads — amber above warn_utilization (80%), and a ⚠ BALANCE OVERSTRESS status-bar warning when exceeded.

Channels tab

Per-channel table: AI number, name, enabled, terminal configuration, requested range (nearest native range is applied), balance flag, and scale/offset/unit for non-balance channels (balance channels are locked to raw volts).

Output & Trigger tab

AO0/AO1 setup (enable, static level, waveform/amplitude/frequency/ offset, start/stop waveform) and the AI start-trigger configuration (mode, source, edge, analog level).

Menus

File: Settings… (Ctrl+,), Save/Load config, Quit. Help: Documentation (this page), About.

6 · Safety & limits

  • Balance overstress warning — live body-frame loads are compared against the .vol file's max loads; utilization above warn_utilization (default 80%) renders amber, and exceeding a limit raises a persistent ⚠ BALANCE OVERSTRESS — reduce load! warning in the status bar. Reduce the aerodynamic load immediately.
  • Raw-volts discipline — recorded bridge data is always raw volts; scale on balance channels is display-only and forced to 1.0, so recorded runs can always be re-reduced with a corrected calibration.
  • AO clamping — commanded output levels clamp to the channel's configured min/max.
  • Input ranges — the requested span maps to the nearest native ± range, so a bridge channel is never digitized on a coarser range than requested.

7 · Troubleshooting

symptomdiagnosis / fix
Connect fails on real hardware Check the NI-MAX alias (Device field vs NI-MAX), the USB link, and that NI-DAQmx is installed. Sim mode requires no install.
Lamp stuck at ARMED, no data Not a fault — a hardware start trigger is configured and the edge has not arrived. Check the trigger source wiring (PFI/APFI pin), edge polarity and analog level, or set the mode back to immediate.
Forces read zero / wrong Load the correct .vol calibration, confirm the balance layout (Force vs Moment names), verify the excitation readback channel, and re-tare with the model unloaded.
Force/Moment switch renamed my channels By design — the layout switch is a pure rename of the four bridge channels at the same AI positions; tares and ring-buffer keys follow the rename.
Overstress warning A body-frame component is past the balance max load — reduce load; check for a stale tare or wrong .vol if it appears at wind-off.
