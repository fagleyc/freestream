DaqBook/2000 — Tunnel Conditions DAQ

User documentation for the daqbook_2000 package — Python driver + dark-mode PyQt6 GUI for the IOtech DaqBook/2005.
Author: C. Fagley — casey.fagley@afacademy.af.edu

1  Overview & purpose

The IOtech DaqBook/2005 at the USAFA subsonic tunnel digitizes the tunnel's dynamic pressure (Pdiff), total pressure (Ptot) and temperature (Temp) transducer voltages. This package is a fully self-contained driver + GUI; the AeroVIS suite composes it with the other instruments (e.g. serving tunnel q via DaqbookAuxSource), but no device package imports another.

It talks to the hardware through the vendor DaqX API (DaqX64.dll, ctypes) over Ethernet. Acquisition is continuous (DaamInfinitePost, immediate trigger) into a circular driver buffer; a poll thread lifts out new scans, converts counts → volts → engineering units and timestamps them against the actual ADC clock. The driver owns its ring buffer (device.ring), so the GUI, the balance app's aux source and the AeroVIS sync layer all read one stream.

2  Hardware & connection

Device aliasDaqBook2005
IP192.168.1.125 (serial 808713)
CH0 Pdiffdifferential, 0..+3 V requested → native 0–5 V (×2)
CH2 Ptotdifferential, ±10 V (×1)
CH4 Tempsingle-ended, 0–10 V (×1)

Default engineering slopes come from Streamlined's PRESSLOPvxi18.PCF: transducer [220] Pdiff 0.386949 psi/V, [690] Ptot 1.92604 psi/V. The temperature transmitter is 1 V = 10 °C (scale 10, unit degC). Default scan rate is 200 Hz.

Hardware constraint (DaqX error 134): single-ended channels accept only bipolar ranges; unipolar is differential-only. The driver's range picker enforces this (an SE 0–10 V request becomes ±10 V native).

One-time DaqX setup (per PC)

daqOpen addresses the device by an alias, not by IP. Two pieces must exist on the PC:

  1. The device alias — create/list/delete with the bundled tool (uses the DLL's own daqCreateDevice): 
    python daqbook_setup.py list
python daqbook_setup.py create DaqBook2005 192.168.1.125
  2. The DaqX Ethernet kernel driver (DaqX600e.sys) must be installed and loaded. Status: sc query DaqX600e.

Fresh Windows 11 PC bring-up (each step was required once)

  1. Windows Driver Policy blocks the 2006 IOtech kernel drivers (CodeIntegrity Event 3077). Remove it with devices\remove_driver_policy.ps1 (Secure Boot round-trip; see script header). Re-check after major Windows updates with devices\check_daqx_driver.ps1.
  2. Kernel driver for this device family is daqbk2k.sys (not just DaqX600e): copy from Drivers\Ethernet_x64 to System32\drivers, sc config daqbk2k start= auto, sc start daqbk2k.
  3. Device alias: python daqbook_setup.py create DaqBook2005 192.168.1.125. Config lives at HKLM\SYSTEM\CurrentControlSet\Services\Daqx\DaqBk2k0.
  4. Registry ACL: daqOpen demands Read/Write on that key, so non-elevated apps get "alias not found" (error 113/−1). One-time fix (elevated): grant BUILTIN\Users FullControl on HKLM\SYSTEM\CurrentControlSet\Services\Daqx (inheritable).
  5. Single-session device: if daqOpen fails with DerrNotOnLine (2) and the device RSTs TCP port 50001, another client (or a stale session) holds it — the embedded status page at http://192.168.1.125/ shows Opened; use its Reset Device link (or power cycle).

The DLL is found automatically in the rig's LabVIEW folder; override with --dll or File → Settings.

3  Getting started

# from the devices/ directory
python run_daqbook_2000_app.py --sim   # pure simulation — no hardware, no DLL
python run_daqbook_2000_app.py         # real device (after DaqX setup)
python probe_daqbook.py                # safe read-only probe (DLL → daqOpen → 1 s)

First-connect checklist

  1. Kernel drivers loaded (sc query DaqX600e, sc query daqbk2k) and alias created.
  2. Nobody else holds the session (http://192.168.1.125/ not Opened).
  3. Launch, verify the alias and rate, press Connect — the lamp shows ACQUIRING @ 200 Hz.
  4. Sanity check (tunnel off, live-verified 2026-07-06): Pdiff ≈ 0 psid, Ptot ≈ 11.38 psia (correct ambient for USAFA's elevation — validates the [690] slope), Temp ≈ 2.4 V raw and noisy (±0.19 V).

Live tab

  • Stat tiles — one per enabled channel (Pdiff, Ptot, Temp) in engineering units, smoothed over the tile-averaging window.
  • Window combo (10 s / 30 s / 2 min / 5 min).
  • Show raw volts — switches the history from engineering units to raw volts.
  • Pause / Resume — freezes the plot (acquisition continues).
  • History plot — full-rate channel histories, envelope-decimated so long windows and high rates stay smooth.

Channels tab

Per-channel table: enable, hardware channel number, name, differential/single-ended, requested voltage range (min/max), engineering scale (units per volt), offset and unit. Changes take effect on the next Connect. The range picker snaps requests to the nearest native range and enforces the SE-bipolar-only constraint.

Serving tunnel q to other consumers (AeroVIS)

from daqbook_2000 import Daqbook2000, DaqbookConfig, DaqbookAuxSource
dev = Daqbook2000(DaqbookConfig())
dev.connect(); dev.start()
aux = DaqbookAuxSource(dev, own_device=True)   # q in Pa from Pdiff psid

DaqbookAuxSource exposes the live stream through a tiny duck-typed interface (dynamic_pressure() -> Pa, temperature_k() -> K) so panels can consume tunnel conditions without cross-package imports.

6  Safety & session care

  • This app commands no motion — it is measurement-only.
  • Single-session device: never kill a Python process mid-acquisition without disconnect() — always the GUI Disconnect or Ctrl-C so the session closes. A wedged session is cleared from the device status page (http://192.168.1.125/ → Reset Device) or by power cycle.
  • Keep the kernel-driver policy check (check_daqx_driver.ps1) in the post-Windows-update routine.

7  Troubleshooting

SymptomCause / fix
daqOpen fails: alias not found (error 113/−1) Alias missing (daqbook_setup.py create) or the registry ACL blocks non-elevated write access — grant BUILTIN\Users FullControl on HKLM\...\Services\Daqx once, elevated.
daqOpen fails: DerrNotOnLine (2), TCP 50001 RSTs Another client or a stale session holds the device — status page shows Opened; use Reset Device or power cycle.
DaqX error 134 configuring a channel Unipolar range requested on a single-ended channel — SE accepts bipolar only. The driver's range picker enforces this; check custom scripts.
Kernel driver won't load (CodeIntegrity Event 3077) Windows 11 driver policy blocks the 2006 IOtech drivers — run devices\remove_driver_policy.ps1 (Secure Boot round-trip), then verify with devices\check_daqx_driver.ps1. Re-check after major Windows updates.
DLL not found It is normally auto-found in the rig's LabVIEW folder; set an explicit path via --dll or File → Settings.
Temp reads noisy Live-verified behaviour: ≈2.4 V raw ±0.19 V with the tunnel off. Set the transmitter scale and consider filtering when known.
Full end-to-end check python verify_daqbook_live.py — the 2026-07-06 run logged 19,917 scans / 20 s at the 1 kHz ADC clock, monotonic 1 ms timebase, zero overruns.
