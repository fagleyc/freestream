Heise PM Indicator

User documentation for the heise package — driver + PyQt6 dark-mode GUI for the Heise PM digital indicator: live pressure and temperature over the RS-232 remote protocol.

1 · Overview & purpose

The Heise PM indicator carries up to two sensor ports — each either a pressure sensor or an RT-1/RT-2 RTD (temperature) module. This app polls the indicator's remote protocol (? query, default 4 Hz), publishes one scan per reply under the configured channel names (default Pressure and Temperature), and exposes the instrument functions: remote pressure-unit selection, zero, tare, damping and battery check. It follows the suite's standard driver shape (connect/start/stop/disconnect, status callbacks, device-owned ring buffer) and is embeddable in host suites.

All protocol facts come from the instrument manual (manual-pm-digital-indicator-im.pdf, §13 and Appendix A — "Indicator serial port functions V1.0").

2 · Hardware & connection

Serial link

  • 300 / 600 / 1200 / 2400 / 4800 / 9600 baud, 8 data bits, 1 stop bit, no parity, no flow control, straight-through cable.
  • Lines end with a selectable end-of-message character — configure the indicator for CRLF. Outgoing commands are CR-terminated; the reader tolerates CR, LF or CRLF and reassembles lines that straddle the read timeout.
  • Set the indicator's interface to the REMOTE protocol (shift → Output → "remote" → baud → EOM).
  • DTR/RTS are asserted at open (some level shifters / USB adapters sit dead until they are high).

3 · Getting started

python run_heise_app.py --sim           # simulator, no hardware
python run_heise_app.py --port COM5     # real indicator
python run_heise_app.py                 # use Search in the GUI
python -m pytest tests\test_heise.py tests\test_heise_gui.py

First-connect checklist

  1. Put the indicator in REMOTE protocol mode (shift → Output → remote → baud → EOM = CRLF).
  2. Click Search… — every COM port is probed with the read-only ? query and the port where a Heise answers is selected (with the detected baud).
  3. Connect — the driver probes with ?, then (by default) pushes the configured pressure units via EUNIT.
  4. Confirm the tiles track the instrument display; Zero Pressure with the reference condition applied if needed.

4 · GUI guide

Connection bar

Port combo + Search… · Baud · Simulate · Connect/Disconnect · status lamp (DISCONNECTED / SIMULATION / LIVE).

Live tab

  • Port tiles — big-number readout per enabled port with its unit.
  • Instrument row — pressure-unit selector (live EUNIT change when connected), Zero Pressure (with confirmation), damping selector (0 off … 3 high), battery-voltage readout.

History tab

One stacked plot per enabled port (the units differ, so they never share a y-axis), time-linked for pan/zoom, showing the last ~2 minutes from the device-owned ring buffer.

Menus

File: Save/Load config, Exit. Help: Documentation (this page), About.

6 · Robustness & watchdogs

  • Serial watchdog — max_consecutive_errors consecutive poll failures → "check cable/power" status and disconnect. Each failure resyncs the line (settle, then drain) so leftovers never poison the next transaction.
  • Line discipline — pyserial's read_until returns partial bytes on timeout (lesson learned live on the sting drives); the reader accumulates until a proper terminator arrives and rejects incomplete responses.
  • Error replies — an ErrNN reply raises a typed HeiseError with the offending command; setters require the OK acknowledgement.
  • Value mapping — the indicator only transmits values for active ports; the reply is mapped onto enabled ports in port order and a count mismatch is an error, not a silent misassignment.
  • Read-only search — the COM scan uses only the ? query; searching is refused while connected (the open port would answer the probe).

7 · Troubleshooting

symptomdiagnosis / fix
"no response (timeout) — is the indicator in REMOTE protocol and powered on?" The indicator must be in REMOTE protocol mode at the matching baud (shift → Output → remote). Check cable (straight-through) and power.
Connect fails with "not responding on COMx" Wrong port or baud — run Search…, which probes every port at the candidate bauds and reports per-port results in the combo tooltips.
Only one value in the reply for two ports Only active ports transmit. Set the unused port's role to off in the config (or activate the second port on the instrument).
Unit change rejected Unit changes apply to pressure-role ports only; RTD ports ignore the pressure code — their unit is chosen on the instrument.
Values freeze / watchdog trips Cable, adapter or power dropout — the watchdog disconnects after 5 consecutive errors. Reconnect after fixing the link; the ring buffer retains history.
