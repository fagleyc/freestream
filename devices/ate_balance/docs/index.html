ATE External Balance — TMS Client

User documentation for the ate_balance package — Python TMS client + dark-mode PyQt6 GUI for the ATE 6-component underfloor balance.
Author: C. Fagley — casey.fagley@afacademy.af.edu

1  Overview & purpose

The ATE Aerodynamic Test Equipment 6-component underfloor (external) balance at the USAFA wind tunnel is controlled by its OGI control PC. This package is a standalone client ("TMS") speaking the OGI's TCP/UDP protocol: live wind-axis loads, motion control (yaw / incidence drives), balance lock/tare and dwell-averaged run points, in a GUI styled to match the Streamlined project.

The app shows raw wind-axis loads only — coefficient reduction (reference geometry, air density) is owned by the Freestream suite. The reduced layer uses Streamlined's exact field names (lift_forces/drag_forces/… , TunnelConditions(Q, …)) so a run drops straight into the Streamlined tooling. The driver mirrors the wtdaq DAQbook2000 driver shape so it can later drop into the AeroSENSE/wtdaq framework unchanged.

Protocol source of truth: AID-010-10015-1.pdf §6 "Client Communication Facilities", cross-checked against OGI/Source/USAFA_Code.pas.

2  Hardware & protocol

ChannelTransportDefault port DirectionPayload
TMSC controlTCP3040 OGI dials the client {M<serial>:CMD args} / {R<serial>:reply}
TMSD dataUDP3041OGI → client b"LOADS" + 6×float32 (+ int32 sync), big-endian
OGIT triggerUDP3042client → OGI TMS_CONNECT

The OGI control PC lives at rig static IP 192.168.1.60. Loads are wind-axis, order Lift, Pitch, Drag, Side, Yaw, Roll (N, N·m). Control commands: ZERO, TAKE_SAMPLE, LOCK_BAL, UNLOCK_BAL, GET_LOCK_STATUS, GET_POSITIONS, GOTO_YAW_POS, GOTO_INC_POS, GET_FILTERS, STOP_ALL_MOTION.

Rig-verified (2026-07-06, probe_ate_rig.py): TMS_CONNECT → OGI dialled back in <10 ms; LOADS stream 300 Hz, 29-byte packets (no sync word) — the USAFA-build variant (device.last_had_sync == False); GET_FILTERS → 300Hz 9 ×6 channels; positions/lock queries answered; unloaded loads ≈0 (|mean| < 0.1, sd ≈ 0.02–0.17), consistent with N/N·m. Still to confirm at the rig: apply a known weight to lock in the N / N·m units assumption.

Drives & span configuration

The rig has two physical drives — YAW (−90..90°) and INCIDENCE (−10..45°). How the logical α/β axes map onto them is the model span configuration:

  • Full span (default): α = incidence drive (GOTO_INC_POS), β = yaw drive (GOTO_YAW_POS).
  • ½ span: semispan model on the turntable — α = the YAW drive (angle of attack comes from yaw); the incidence drive is unused and there is no β axis.

3  Getting started

pip install -r requirements.txt          # numpy, PyQt6

# 1) Pure simulation — no hardware, no network:
python run_ate_balance_app.py --sim

# 2) Against the bundled Python emulator (two terminals):
python -m ate_balance.emulator --tms-ip 127.0.0.1
python run_ate_balance_app.py --ip 127.0.0.1

# 3) Against the real OGI on the rig network (192.168.1.60, the default):
python run_ate_balance_app.py

# read-only rig check (safe queries only):
python probe_ate_rig.py

First-connect checklist

  1. By default the client listens on TCP 3040 — the OGI dials in, per the manual. Press Connect; the lamp goes WAITING FOR OGI → LINKED once the OGI connects and the LOADS stream arrives.
  2. If the OGI doesn't dial in, press Trigger (sends TMS_CONNECT on the OGIT port to prompt a (re)dial). Auto-trigger on Connect is on by default.
  3. Check the stream rate readout (~300 Hz on the rig) and that unloaded loads read near zero.
  4. Testing with OGI_Sim.exe (the rig's own simulator): point its OGI.ini [TMSC] IP at the machine running this client, start it, and run with --ip set to the OGI_Sim host (127.0.0.1 if on the same PC).

Live tab

  • Wind-axis load bars — the six loads (Lift, Drag, Side, Roll, Pitch, Yaw), smoothed over the bar-averaging window, one fixed color per channel across all panels.
  • Overstress hint — shown while a smoothed load exceeds a nonzero rated maximum (Settings → Rated load maxima).
  • Device status strip — yaw/incidence positions, stream rate, sync input, bar smoothing, and tunnel q (visible only when a real DaqBook aux source is attached via --daqbook).

Motion tab

  • Model span combo — the single editor for span_config; relabels/disables the drive boxes so the α/β ↔ drive mapping is unmistakable, and is recorded into data files.
  • Yaw drive / Incidence drive boxes — live position, target spin box clamped to the drive limits (yaw −90..90°, incidence −10..45°), Move, Stop (stops all motion) and To Zero.
  • Balance box — Lock / Unlock / Status, Zero (tare), and E-STOP (STOP_ALL_MOTION).

Run tab

  • Single sample (OGI average) — duration + Take Sample sends TAKE_SAMPLE; the OGI's averaged SAMPLES/TARES replies show in the "Last sample" line.
  • Dwell point (live average) — enter α/β, Start/Stop Dwell: frames are averaged live into a reduced point (mean ± 1σ tooltips) appended to the results table.
  • Time history — full-rate plots of every frame in the ring buffer (300 Hz on the rig); window 5/10/30/60 s, Pause/Resume.
  • Results table + export — Clear, Export CSV…, Export TestCase (.npz)… (Streamlined-shaped named arrays).

Menus

6  Safety & interlocks

  • E-STOP (Motion tab) and the per-drive Stop buttons all issue STOP_ALL_MOTION to the OGI.
  • Drive limits: target spin boxes are clamped to the documented drive travels (yaw −90..90°, incidence −10..45°).
  • Balance lock: Lock / Unlock / Status via LOCK_BAL / UNLOCK_BAL / GET_LOCK_STATUS — lock the balance before model changes.
  • Rated load maxima: configure under File → Settings → "Rated load maxima"; the Live panel flags OVERSTRESS while a smoothed load exceeds a nonzero maximum.
  • In the ½-span configuration the incidence controls are disabled — the drive is unused and cannot be commanded accidentally.

7  Troubleshooting

SymptomCause / fix
Lamp stuck at WAITING FOR OGI The OGI hasn't dialled in: press Trigger (TMS_CONNECT). Check the OGI's OGI.ini [TMSC] IP points at this machine, and that the local firewall allows TCP 3040 / UDP 3041 inbound.
Linked but no loads The LOADS stream is UDP on 3041 (OGI → client) — verify the port and firewall. On the rig the stream is 300 Hz, 29-byte packets without the sync word (device.last_had_sync == False) — both variants are decoded.
Which machine does the OGI dial? The address in its own OGI.ini [TMSC] IP. AteConfig.from_ogi_ini() can seed the client config from that file.
Loads magnitudes uncertain Units are N / N·m per the protocol; the README notes the final known-weight check on Lift is still to be performed at the rig. probe_ate_rig.py re-runs the whole read-only verification safely at any time.
Testing without the rig Three options: --sim (pure synthetic), python -m ate_balance.emulator (real sockets, FakeOGI), or the rig's own OGI_Sim.exe.
Coefficients missing By design — this device measures loads only; coefficient reduction (geometry, density) is owned by the Freestream suite.
