LSWT Sting Drive

User documentation for the lswt_sting package — driver + PyQt6 dark-mode GUI for the LSWT model sting: dual-axis (alpha/beta) serial stepper-indexer drive, protocol recovered from the deployed C# Tool_LSWT_Sting/Core.exe.

1 · Overview & purpose

Two SLO-SYN/Compumotor-style serial stepper indexers are daisy-chained on one RS-232 port (9600-8N1, CR-terminated), one per axis:

AxisUnitSteps/degreeAccel/Decel Velocity
Alpha12741.052510.8528 0.108
Beta266.820.032

Commands are <unit><cmd> (1R, 2G…); moves are relative distances in motor steps (D then G); position is the indexer's step counter (PR), zeroed with PZ. The position loop runs INSIDE the indexers — the host loads a distance, starts the move, and polls status until READY.

Open loop — no encoder, no brake. Angle is tracked by the indexer step counter. The counter zero is established by the operator: physically set the sting to a known angle, enter it, and Zero the axis (PZ). Until an axis is zeroed, absolute moves are refused; only jog steps are allowed. Angle = zero_offset_deg + counts / steps_per_degree.

2 · Hardware & connection

Wire format (recovered from the deployed C#)

  • 9600 baud, 8N1, no handshake; lines terminated with a bare CR (\r); 500 ms read/write timeouts. DTR/RTS are asserted at open (some RS-232 level shifters and Prolific USB adapters sit dead until they are high).
  • Unit 1 = Alpha, 2 = Beta, empty unit = broadcast (SSI1, FSD1).
  • The drives echo every command line; the echo is read back and validated after each write (an unexpected echo means a wiring/addressing problem).

Command set

commandmeaning
A / AD / V acceleration / deceleration / velocity (drive units, rev/s and rev/s²; transmitted byte-for-byte as the legacy tool did)
D<steps> + G load a relative distance, go
Sstop (immediate; the E-stop path writes it raw with no echo wait)
Zdrive reset (bring-up; drives re-boot ~1.1 s after it)
PZ / PRzero / report the step counter (signed)
Rstatus request → *B busy · *R ready · *S stalled
LD3disable the drive's limit inputs (as the legacy tool does)
SSA0 / SSI1 / FSD1 interface setup, sent verbatim (and BLIND — these switches change the drives' echo behaviour as they execute)

Bring-up sequence (mirrors the legacy InitHw)

Probe with 1R → broadcast SSI1 → per axis SSA0 (+ optional Z reset, then a ~1.1 s re-boot wait) → wait for READY (stall at init is fatal: power-cycle the drives) → LD3 → read positions → push A/AD/V → broadcast FSD1.

Direction signs (field-verified 2026-07-22)

counts = direction × (angle − zero_offset) × steps_per_degree. Alpha = −1 (positive angle = negative counts; the Core.exe IL agrees), Beta = +1 (observed backwards with −1; the legacy MoveStingByDegrees Beta branch, which omits the negation, had the physically correct sign — its absolute path carried a dormant sign bug). Getting a sign wrong drives the axis into the hard stop (Alpha stall, 2026-07-22).

3 · Getting started

python run_lswt_sting_app.py --sim   # wire-level drive emulator
python run_lswt_sting_app.py         # real drives on the configured COM
python -m pytest tests\test_lswt_sting.py tests\test_sting_safety.py

Use Search… on the connection bar to probe every COM port with a read-only 1R status query and auto-select the one where the sting chain answers. Sim mode swaps in SimSerial, a wire-level emulator, so the full GUI runs without hardware.

First-connect checklist

  1. Confirm the sting is physically in a safe position — connect runs the bring-up sequence, and with Send Z at connect enabled the drive reset MAY CAUSE UNCONTROLLED MOVEMENT (legacy warning).
  2. After connect, each axis shows "— not zeroed —": physically read the sting angle, then Set Current Angle… per axis (declares the angle and issues PZ).
  3. Verify a small step jog moves the expected direction before any absolute Go.
  4. Soft limits ship as a guess (Alpha −15…+30°, Beta −15…+15°) — the legacy limits live in the rig's "Tunnel Default File"; confirm on first live run (Limits tab).
Position restore: the live angle is checkpointed continuously to sting_state.json and marked clean on orderly disconnect. On reconnect the zero reference is restored so an abrupt shutdown does not lose the position — with a loud warning to verify physically, since an unpowered sting can droop.

4 · GUI guide

Connection bar

Port combo + Search… · Simulate · Connect/Disconnect · status lamp · FAULT badge (lights red with the fault text as tooltip) · STOP ALL — immediate stop of both axes, no confirmation, always live while connected.

Motion tab

  • Alpha | Beta axis boxes: big angle readout (or "— not zeroed —" with steps), state label (idle / → target / MOVING), absolute Target + Go (unlocked once zeroed), Stop, Step −/+ jog with a configurable °/step size, and Set Current Angle… (declare the physical angle, confirm, zero the counter).
  • Go Both: α + β targets moved together (requires both axes zeroed).
  • Drive box: Reset Fault (clears the latched fault after the cause is addressed) and Reinitialize Drives (re-runs the bring-up incl. the Z reset — warns about uncontrolled movement and requires re-zeroing both axes).

History tab

Motion indicator + angle-vs-time plot for both axes.

Limits tab

Soft travel limits per axis (take effect immediately), park behaviour (Park Alpha on disconnect + park angle), and comms settings for the next connect (send Z at connect, poll period).

Menus

File: Settings… (Ctrl+,), Save/Load config, Quit. Help: Documentation (this page), About.

6 · Safety & interlocks

The sting has no brake. Park is the rest position; parking on disconnect is ON by default. The live angle is checkpointed continuously, and an interpreter-exit hook parks and saves as a last line of defence.
  • STOP ALL / E-stop — writes 1S/2S immediately from the calling thread, no echo wait, no queueing behind the poll loop; the echoes are drained atomically so a concurrent poll can never mistake them for its own response.
  • Soft travel limits — enforced host-side before any command is sent; absolute moves additionally require the axis to be zeroed.
  • Stall / timeout fault latch — a *S (stall) response or a move timeout latches a FAULT: everything is stopped and further motion refused until Reset Fault. A stall additionally requires the drives to be power-cycled (legacy behaviour) and the axes re-zeroed.
  • Reinitialize — re-runs the bring-up sequence including the Z reset; the legacy tool warns the reset MAY CAUSE UNCONTROLLED MOVEMENT if the sting is not in a safe position, and both axes must be re-zeroed afterwards. The GUI requires an explicit confirmation.
  • Serial watchdog — max_consecutive_errors failures in the poll loop → stop everything, drop the connection, report via the status bar.
  • Zeroing gate — un-zeroed axes only allow step jogs; the absolute Go/target stays locked.

7 · Troubleshooting

symptomdiagnosis / fix
No response at connect ("controllers are not responding") Check drive power and the COM port (use Search…). Some USB adapters/level shifters need DTR/RTS high — the driver asserts them, but a dead adapter still probes silent.
Echo timeouts during bring-up Normal for SSA0/SSI1/Z: these switches change the drives' echo behaviour as they execute (seen live on COM9 via a Prolific adapter). The driver sends them blind, like the legacy tool. Persistent echo mismatches on ordinary commands indicate a wiring/addressing problem.
FAULT badge: STALL The motor could not follow the commanded profile (e.g. driven into the hard stop — this is exactly what a wrong direction sign does). Address the cause, power-cycle the drives (legacy requirement), Reset Fault, and re-zero both axes.
Axis moves backwards The axis direction sign is wrong. Field-verified defaults: Alpha −1, Beta +1 (2026-07-22). Do not trust the legacy C#'s absolute Beta path — it carried a dormant sign bug.
Garbled/fragmented responses while a drive is stepping Known behaviour: responses can straddle the read timeout ('*+0' / '000000000' fragments seen live). The protocol reassembles lines and resyncs after failures; if a transaction still fails, the next one starts clean.
Position wrong after a crash/power loss The restore is open loop — the checkpoint warns to verify physically; an unpowered sting can droop. Re-zero against the physical angle if in doubt.
Absolute Go greyed out The axis is not zeroed (or a fault is latched). Set Current Angle… first; Go Both needs BOTH axes zeroed.
