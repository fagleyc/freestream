ARC Crescent — SSWT Sting Drive

User documentation for the ac_delta package — Python driver + dark-mode PyQt6 GUI for the crescent's dual Delta C2000 drives.
Author: C. Fagley — casey.fagley@afacademy.af.edu

1  Overview & purpose

The ARC crescent is the model-attitude drive of the USAFA SSWT sting: two Delta C2000 variable frequency drives, one per axis (Alpha and Beta), each reachable over Modbus TCP. This package replaces the deployed C# tooling (FTool_SSWT_Sting / HwControllerStingSSWT) with a standalone Python driver and GUI.

Motion is host-closed-loop: the PC reads the encoder, picks one of five discrete speed steps from the remaining distance, writes the step to the drive, and issues a stop inside the tolerance band. The package sits in the AeroVIS / Freestream instrument ecosystem as the sting attitude device; the GUI panel (CrescentPanel) is also embeddable in host suites, sharing a single live drive.

Improvements over the C# implementation (from the package README): persistent Modbus connections (the C# reconnected before every read/write), change-only step writes, a 50 ms loop (C#: 100 ms), configurable deceleration bands, true synchronous dual-axis moves, and layered safety (soft limits, per-axis Stop, global E-STOP, comm watchdog).

Register map (verified live 2026-07-06)

Register (0-based wire address)Meaning
8713Encoder, signed 16-bit, axis-unique. The C# reads "8714" through FieldTalk, which counts register references from 1 — pymodbus is 0-based, so the wire address is 8713. (8714 itself is a constant 0, which is why the first driver build read zeros.)
8192 (0x2000)Delta C2000 control word (LabVIEW-confirmed; manual page in info/). Bits: 1–0 run/stop (01=stop, 10=run), 5–4 direction (01=FWD, 10=REV), 7–6 accel/decel set, 11–8 step-speed 0–15, 12 enable. Written as "command register 8193" in the C# (FieldTalk 1-based).
0x2105DC bus voltage telemetry (read 370.0 V live).
0x220EIGBT temperature telemetry (read ≈23 °C live).

Command words

The C# "step values" are full control words (e.g. 0x1112 = run + FWD + step 1 + enable):

Speed step12345
Forward43704626 48825138 5394
Reverse43864642 48985154 5410
Stop17 (Alpha) / 33 (Beta) — both plain stops; the direction bit differs and is harmless.
Single-register reads only. Block reads spanning undefined C2000 registers fail whole-block with Illegal Data Address — scan with count=1.

Angle calibration

Two-point form, identical to the C#:

angle = angle_high − (encoder_high − encoder) / clicks_per_degree

The measured on-rig slopes (two-point calibration, 2026-07-06) are the package defaults: Alpha 294.8292, Beta 202.9586 clicks/degree. The offset still needs a limit-switch re-zero each setup, so axes start UNCALIBRATED until routine 2 (single-point re-zero) is performed. The original constants live in the rig's XML tool config.

The LabVIEW speed-band logic works in encoder counts: remaining >500 → step 5, >400 → 4, >200 → 3, >100 → 2, else 1 — convert with clicks/degree when tuning the degree bands in Settings.

3  Getting started

# from the devices/ directory
python run_ac_delta_app.py --sim    # physics sim, no hardware
python run_ac_delta_app.py          # real drives (Alpha .11 / Beta .12)
python probe_crescent.py            # READ-ONLY live check (no motion)

First-connect checklist

  1. Confirm the rig network is reachable: Alpha 192.168.1.11, Beta 192.168.1.12, Modbus TCP port 502.
  2. Run probe_crescent.py first — it reads encoders and telemetry without commanding any motion.
  3. Launch the GUI, check the IPs in the Connection bar, and press Connect. The lamp shows LIVE (or SIMULATION with the Simulate box checked).
  4. Uncalibrated axes show raw encoder counts and are jog-only. Calibrate (Calibration tab) before trusting angles or using angle moves.
Uncalibrated jogs have no limit protection — watch the hardware.

Motion tab

Axis cards (Alpha, Beta) — one per drive:

  • Big readout: angle in degrees when calibrated; raw encoder counts (with an "UNCALIBRATED — jog only" note) otherwise. The state label shows idle, JOG or the active target (→ +12.50°).
  • Target + Move: closed-loop angle move — enabled only when the axis is calibrated and connected; the spin box is clamped to the soft limits.
  • Stop: stops that axis immediately.
  • Jog − / Jog +: hold-to-run — press starts motion, release stops it. While calibrated, jogs auto-stop at the soft limits.
  • Speed 1–5: single-click speed-step selector used by jogs (default step 2).

Synchronous move box: enter α and β targets and press Move Both — both axes start in the same control tick and each brakes on its own remaining distance. Enabled only when both axes are calibrated. The E-STOP button below it stops all axes immediately (see Safety).

Angle history plot: both axes on one time axis (seconds before now). Shows angle when both axes are calibrated, raw encoder counts otherwise; redraws are envelope-decimated so long windows stay smooth.

Calibration tab

One group per axis, each with the live encoder readout and three ways to calibrate:

  1. Full cal (two points): jog to a known position, enter its angle, Capture pt 1; jog to a second known position, enter its angle, Capture pt 2 → compute. Slope (clicks/degree) and offset both come from the pair.
  2. Re-zero (single point): with the slope already known (the measured rig defaults), jog onto a reference (e.g. the limit switch), enter its known angle, Set position = angle. Only the offset moves.
  3. Constants row: view or directly enter clicks/deg, angle_high, encoder_high and Apply (e.g. values from the rig's XML tool config).

6  Safety & interlocks

  • E-STOP: the red button on the Motion tab stops all axes immediately (not queued).
  • Per-axis Stop on each axis card.
  • Soft travel limits are checked before any motion command; calibrated jogs auto-stop at the limits.
  • Modbus watchdog: on max_consecutive_errors consecutive comm failures, all axes are stopped and the failing axis is dropped.
  • Uncalibrated axes are jog-only — angle targets unlock per axis once calibrated; the synchronous move only when both are. Uncalibrated jogs have no limit protection.

7  Troubleshooting

SymptomCause / fix
pymodbus "transaction_id" errors in the log The C2000 PLC occasionally answers late; pymodbus skips the stale frame and recovers. The driver retries the read once and silences that library log — real failures still surface through the drive watchdog.
Encoder reads zero Reading the FieldTalk-numbered register 8714 instead of wire address 8713 (the driver already uses 8713 — check any custom scripts).
Block register reads fail with Illegal Data Address Expected: block reads spanning undefined C2000 registers fail whole-block. Scan single registers (count=1).
Angles look wrong / offset drifts between setups The offset needs a limit-switch re-zero each setup (Calibration tab, routine 2). The GUI flags UNCALIBRATED and shows raw counts until then.
Moves brake too early/late vs. the LabVIEW profile Tune File → Settings: loop period, decel bands and max step are all live-adjustable. LabVIEW bands are in encoder counts (500/400/200/100) — convert with clicks/degree.
Is it really the right device? C2000 telemetry confirms it: 0x2105 DC bus 370.0 V, 0x220E IGBT ≈23 °C. The RedLion .cd3 is encrypted; not needed.
